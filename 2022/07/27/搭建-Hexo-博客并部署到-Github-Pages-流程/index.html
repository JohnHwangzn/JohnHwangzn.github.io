搭建 Hexo 博客并部署到 GitHub Pages 流程

0. 前言

Hexo 是一个静态博客（纯 HTML 格式）框架，使用 Markdown 解析文章，支持一键部署到 GitHub Pages、Heroku 等静态网站托管平台上。Hexo 是目前最主流的静态博客框架之一，同类型的框架还有 Hugo、jekyll。

关于这三款框架的对比，可以参见 Jekyll / Hugo / Hexo Comparison | The Coding Notes (lexcao.io)。

1. 搭建步骤

Hexo 依赖以下两个应用程序：

  • Git
  • Node.js

2. Hexo 的基本操作

2.1 Hexo 工程的目录结构

各文件/目录的功能说明如下：

  • _config.yml : 可以进行以下设置：网站信息、网址结构、目录配置、文章属性、分类&标签、日期/时间格式、分页、扩展、处理或忽略某些目录、使用替代配置文件等。
  • package.json : 应用程序的信息。每个 JavaScript 项目（无论是 Node.js 还是浏览器应用程序）都可以被当作 npm 软件包，并且通过  package.json 来描述项目和软件包信息。
  • scaffolds : 模版目录。当您新建文章时，Hexo 会根据 scaffold 来构建文件。模板指的是在新建的文章文件中默认添加的内容。
  • source ：存放用户所有文件的地方。
  • themes ：主题目录。Hexo 会根据主题来生成静态页面。
  • public ：当 hexo 生成静态页面时，在 source 目录下的 Markdown 和 HTML 文件会被解析并放到 public 目录下，而其他文件会被直接拷贝过去。注意，除 _posts文件夹之外，开头命名为 _ (下划线)的文件 / 目录和隐藏的文件将会被忽略。

2.2 创建新的文章/页面

创建一篇新的文章或页面：

布局（layout）

hexo 的三种默认的布局：post、page 和 draft。这三种不同类型的文件，在被创建时，会被保存到不同的路径，对应的保存路径如下：

  • post: source/_posts
  • page: source
  • draft: source/_drafts

草稿（draft）

如果新建了一个草稿，那么它将会被存储在 source/_drafts 下。草稿默认不会显示在页面中，不过可以在执行时加上--draft 参数来预览草稿。可以通过 publish 命令将草稿移动至 source/_posts 目录下：
